\# Архитектура решения для масштабирования



\## Общее описание



Система предназначена для мониторинга позиций товаров на Ozon. 

В текущей реализации — это консольный скрипт на Python с использованием Playwright.

Для масштабирования до 10 000 проверок в день предлагается следующая архитектура.

\##  Технологии



| Компонент | Технология | Причина выбора |

|-----------|-----------|----------------|

| \*\*API\*\* | FastAPI | Асинхронный, быстрый, встроенная документация |

| \*\*Очередь задач\*\* | Celery | Распределенная обработка, поддержка retry |

| \*\*Брокер\*\* | Redis | Быстрый, надежный, встроенный rate limiting |

| \*\*База данных\*\* | PostgreSQL | Надежность, поддержка JSON, индексы |

| \*\*Парсинг\*\* | Playwright | Современный, антидетект, быстрый |

| \*\*Telegram бот\*\* | aiogram | Асинхронный, удобный, активная поддержка |

| \*\*Прокси\*\* | Rotating Proxy Pool | 50+ IP, ротация каждые 10 запросов |

| \*\*Логирование\*\* | ELK Stack | Централизованные логи, мониторинг |

| \*\*Мониторинг\*\* | Prometheus + Grafana | Визуализация метрик, алерты |

\##  Процесс работы



\### 1. Запрос от пользователя

```json

POST /api/parse

{

&#x20; "query": "наушники беспроводные",

&#x20; "skus": \["3531022625", "3858478191"],

&#x20; "schedule": "daily"

}

2\. Обработка

API валидирует запрос



Создает задачу в Celery



Задача попадает в очередь Redis



3\. Выполнение

Worker берет задачу



Выбирает прокси из пула



Выполняет парсинг через Playwright



Сохраняет результат в PostgreSQL



4\. Алерты

Если позиция < 50 → отправка в Telegram



Если ошибка > 3 раз → уведомление админу



&#x20;Устойчивость и защита

Прокси-пул

Количество: 50+ IP-адресов



Ротация: каждые 10 запросов



Типы: HTTP, HTTPS, SOCKS5



Бюджет: \~$30-50/мес



Rate Limiting

На уровне Celery: 1 запрос / 3 секунды



На уровне прокси: 5 запросов / 10 секунд



На уровне Ozon: не более 20 запросов/минуту с одного IP



Retry-механизм

Максимум попыток: 3



Экспоненциальная задержка: 5s, 10s, 20s



Смена прокси при каждой попытке



Мониторинг

Прометей: метрики (время ответа, ошибки, позиции)



Алерты: если > 10% ошибок или позиция упала ниже 50



Логи: централизованный сбор через ELK

Бюджет на инфраструктуру (в месяц)

Компонент	Вариант	Цена

Сервер (VPS)	2 vCPU, 4GB RAM	$40-60

База данных (PostgreSQL)	Managed RDS	$25-40

Redis	Managed (или на сервере)	$10-15

Прокси-пул	50+ IP с ротацией	$30-50

Мониторинг	Prometheus + Grafana	$10-20

Telegram бот	Бесплатно	$0

Резервное копирование	S3 / Object Storage	$5-10

Итого:		\~$120-195/мес

&#x20;Риски и их снижение

Риск	Решение

Блокировка Ozon	Ротация прокси и User-Agent

Изменение HTML	Мониторинг ошибок парсинга

Капча	Перезапуск браузера со сменой прокси

Медленная работа	Параллелизация (10-20 worker'ов)

Падение сервера	Автоматический перезапуск (systemd)

&#x20;Масштабирование

При 10 000 проверок/день

Запросов: 10 000



Время на запрос: 5 секунд



Общее время: 50 000 секунд (\~14 часов)



Нужно worker'ов: 3-5 параллельно



При 50 000 проверок/день

Запросов: 50 000



Нужно worker'ов: 10-15 параллельно



Дополнительно: шардирование базы данных



&#x20;Безопасность

API: JWT аутентификация



База данных: Шифрование соединения (SSL/TLS)



Прокси: Использование доверенных провайдеров



Логи: Очистка чувствительных данных



&#x20;Итог

Система готова к обработке 10 000 проверок в день.

Устойчивость достигается через прокси-пул и rate limiting.

Бюджет \~$150/мес является реалистичным для старта.
